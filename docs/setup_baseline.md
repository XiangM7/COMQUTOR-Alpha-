# TradingAgents Baseline Setup

## Purpose

This repository is a fork of TradingAgents.

The goal is not to rebuild TradingAgents. The goal is to use TradingAgents as the execution baseline and connect its outputs into COMQUTOR's future Structure Intelligence Engine.

TradingAgents output will later become the input source for:

- Structure Graph
- Conflict Intelligence
- Alpha Memory
- SIC Finance platform

## Start From Zero

Clone the forked repository:

git clone https://github.com/XiangM7/COMQUTOR-Alpha-.git
cd COMQUTOR-Alpha-

Create or activate the Python environment:

conda activate tradingagents

Install the project in editable mode:

pip install -e .

Create a local `.env` file:

touch .env

Add the LLM API key:

ANTHROPIC_API_KEY=your_key_here

Optional Anthropic provider configuration:

TRADINGAGENTS_LLM_PROVIDER=anthropic
TRADINGAGENTS_OUTPUT_LANGUAGE=Chinese
TRADINGAGENTS_MAX_DEBATE_ROUNDS=1
TRADINGAGENTS_MAX_RISK_ROUNDS=1

Load environment variables:

source .env

Start the TradingAgents CLI:

tradingagents

Alternative source command:

python -m cli.main

## Run One Stock Analysis

NVDA

Suggested baseline configuration:

- Ticker: NVDA
- Analysis date: 2026-06-30
- LLM provider: Anthropic
- Analysts: market, news
- Research depth: shallow / 1 round
- Reasoning effort: low or medium for cost control

After the CLI completes, save the generated report.

complete_report.md
1_analysts/*.md
2_research/*.md
3_trading/trader.md

Recommended future artifact directory:

artifacts/tradingagents/<ticker>/<analysis_date>/<run_id>/

Recommended future metadata file:

manifest.json

The manifest should record:

- ticker
- analysis_date
- provider
- quick model
- deep model
- selected analysts
- commit
- run command
- report paths

## Known Issues

1. TradingAgents is interactive by default.
2. Full automation will require a headless runner.
3. LLM API calls may generate cost.
4. FRED macro indicators require `FRED_API_KEY`.
5. Reddit RSS may hit rate limits.
6. High reasoning effort is costly and slow.
7. The two baseline reports are now normalized under the same top-level `reports/` directory.
8. Future structure computation should not depend on manually named report folders.
